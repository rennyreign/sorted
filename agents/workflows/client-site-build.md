# Client Site Build Workflow

## Goal

Deliver clean, modern, practical local-business websites quickly and consistently using the Sorted workflow and Taste Skill ecosystem.

## Workflow

### 1. Create project folder

Create a new folder under `websites/`.

Copy:
- brief.md
- qa.md
- handoff.md
- notes.md

from `websites/_template/`.

## 2. Add source material

Add:
- mockups
- screenshots
- logos
- images
- copy
- brand assets
- references

inside the client folder.

## 3. Fill out the brief

Before build work starts:
- identify business type
- identify primary CTA
- decide Standard or Premium
- list required sections
- define audience

## 4. Build process

Recommended skill stack:

- sorted-local-site-refresh
- image-to-code
- design-taste-frontend
- redesign-existing-projects where relevant
- full-output-enforcement when needed

## 5. Build order

1. responsive structure
2. typography and layout
3. imagery and sections
4. interactions and transitions
5. mobile optimisation
6. metadata and footer
7. QA pass

## 6. QA

Use `qa.md` before presenting work.

## 7. Handoff

Complete `handoff.md` before launch or client delivery.

## 8. Knowledge capture

After delivery, update `notes.md` with:

- reusable sections
- reusable prompts
- reusable animations
- mistakes to avoid
- patterns worth standardising

The goal is compounding delivery speed and consistency over time.
